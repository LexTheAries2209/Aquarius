# Aquarius V1.0.1 Release Notes

Release date: 2026-07-07

Aquarius V1.0.1 is the first public release for QTake, monitor-recording, dailies, and proxy metadata recovery workflows. It reads burned-in clip names, reel names, and source timecode from the picture, then turns those results into DaVinci Resolve metadata or MOV / QuickTime TMCD timecode.

## Highlights

- Import one video, multiple videos, folders, or drag media directly into the window.
- Draw OCR regions on top of the video preview for clip name, reel, and timecode fields.
- Save, rename, import, and export ROI presets as JSON for different QTake overlay layouts.
- Analyze one clip or the full queue.
- Sample multiple frames and classify results as reliable, needs review, or failed.
- Detect source timecode rate automatically, or manually choose 23.976, 24, 25, 29.97 NDF, 29.97 DF, or 30.
- Warn about frame-rate mismatch, suspected drift, high-risk readings, and mismatch against existing TMCD metadata.
- Optionally read MediaInfo CLI output for richer container, codec, frame-rate, duration, and current timecode details.
- Manually override clip name, start TC, reel number, and camera ID before export.
- Export DaVinci Resolve metadata CSV.
- Optionally write recovered start TC into QuickTime TMCD tracks.
- Optionally copy files to a new folder and rename them from recognized or manually entered clip names.
- Optionally create a companion DaVinci CSV after renaming, using the renamed output files as Resolve match keys.

## Download And Install

1. Download `Aquarius-v1.0.1-macOS.zip` from the GitHub Release.
2. Unzip it and move `Aquarius V1.0.1.app` to `Applications` or your own tools folder.
3. On first launch, if macOS Gatekeeper shows an internet-download warning, right-click the app in Finder and choose Open, or allow it from System Settings > Privacy & Security.

## Requirements

- macOS 13.0 or later.
- The release build does not require Xcode.
- Optional dependency: MediaInfo CLI. With it installed, Aquarius can show richer source metadata; without it, OCR, CSV export, copy/rename, and TMCD writing still work.

## Known Limitations

- MOV / QuickTime is the only officially supported format for this release.
- OCR quality depends on readable burned-in text and accurate ROI placement; different overlays usually need separate ROI presets.
- Automatic source TC inference only covers 24, 25, and 30. 23.976, 29.97 NDF, and 29.97 DF must be selected manually.
- 29.97 DF is supported for manual parsing and CSV export. QuickTime TMCD writeback remains protected for non-integer and drop-frame rates.
- Direct source-file TMCD writing only supports MOV / QuickTime files with an existing writable TMCD track.
